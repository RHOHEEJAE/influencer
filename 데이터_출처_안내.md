# 인플루언서 데이터 출처 안내

학습 데이터 수집 및 사업용 인플루언서 선별을 위해 활용할 수 있는 **데이터 출처**와 **주의사항**을 정리했습니다.

---

## 1. 공식 API (권장)

| 플랫폼 | 데이터 종류 | 접근 방법 | 비고 |
|--------|-------------|-----------|------|
| **YouTube** | 채널 구독자, 조회수, 동영상 수, 제목/설명 | [YouTube Data API v3](https://developers.google.com/youtube/v3) | 무료 할당량 있음. API 키만 발급하면 사용 가능. 본 코드에서 예시로 사용 중. |
| **TikTok** | 크리에이터 프로필, 팔로워, 좋아요, 동영상 메타데이터 | [TikTok for Developers](https://developers.tiktok.com/) (Login Kit, Display API 등) | 앱 심사 후 사용. 비즈니스/연구 목적에 따라 제한 있음. |
| **Instagram** | 프로필, 팔로워, 미디어 수, 인사이트 | [Meta Graph API](https://developers.facebook.com/docs/instagram-api) | Instagram Business/Creator 계정 필요. 인사이트는 해당 계정 본인 데이터 위주. |
| **Twitter/X** | 프로필, 팔로워, 트윗 수, 참여 지표 | [X API](https://developer.x.com/) | 유료 플랜 위주. 무료 티어는 제한적. |

- **YouTube**: 가장 쉽게 시작할 수 있어 이 프로젝트에서 기본 예시로 포함되어 있습니다.
- **TikTok / Instagram / X**: 각 플랫폼 개발자 사이트에서 앱 등록 후 이용 조건과 과금을 확인하세요.

---

## 2. 인플루언서 마케팅·분석 플랫폼 (B2B)

사업용으로 **이미 정제된 인플루언서 DB**를 쓰고 싶다면 아래와 같은 서비스를 검토할 수 있습니다. 대부분 **API 또는 CSV/엑셀 내보내기**를 제공합니다.

| 서비스 (예시) | 제공 데이터 | 특징 |
|---------------|-------------|------|
| **HypeAuditor** | 인플루언서 프로필, 팔로워/참여율, 가격대, 사기 지표 | 감사·신뢰도 지표 제공. API/연동 가능. |
| **Upfluence** | 크리에이터 검색, 연락처, 성과 지표 | 브랜드 캠페인 연동. |
| **Aspire** | 인플루언서 검색, 캠페인 성과 | CRM·캠페인 관리와 결합. |
| **Grin** | 크리에이터 검색, 계약·결제 관리 | 엔터프라이즈용. |
| **Influence.co** | 인플루언서 디렉토리, 요금 추정 | 소규모 브랜드에도 사용. |
| **CreatorIQ** | 대규모 크리에이터 DB, 캠페인 분석 | 대기업·에이전시용. |

- 계약 전 **데이터 제공 범위(필드, 건수, 갱신 주기)**와 **API/엑셀 내보내기** 가능 여부를 확인하는 것이 좋습니다.
- 이런 플랫폼에서 내보낸 CSV/API 응답을 이 프로젝트의 `influencer_collector` 스키마에 맞춰 매핑하면 학습 데이터로 활용할 수 있습니다.

## 3. 공개 데이터셋 (학습·연구용)

| 출처 | 설명 |
|------|------|
| **Kaggle** | "influencer", "instagram dataset", "youtube channel" 등으로 검색하면 팔로워·참여 지표가 포함된 CSV/데이터셋을 찾을 수 있음. |

- 라이선스와 수집 시점을 확인한 뒤, **학습용**으로만 사용하는 것을 권장합니다.

## 4. 웹 스크래핑 (주의 필요)

- **공개 프로필 페이지**를 스크래핑해 팔로워 수, 게시물 수 등을 수집하는 방식입니다.
- **각 플랫폼 이용약관(Terms of Service)**을 반드시 확인하세요. 많은 플랫폼이 자동화된 수집을 금지합니다.
- 법적·윤리적 리스크가 있으므로, **가능하면 공식 API나 B2B 플랫폼**을 우선하는 것이 안전합니다.

---

## 5. 이 프로젝트에서의 사용 흐름

1. **YouTube**: `.env`에 `YOUTUBE_API_KEY` 설정 후 `influencer_collector.py` 실행 → `data/influencers_youtube.csv` 생성.
2. **다른 API/플랫폼**:  
   - 해당 API로 수집한 필드를 `normalize_record()` 또는 동일한 CSV 컬럼 구조에 맞춰 매핑.  
   - `save_influencers_csv()` / `save_influencers_json()`으로 저장.
3. **B2B 플랫폼**:  
   - 플랫폼에서 내보낸 CSV/API를 같은 스키마(채널 ID, 사용자명, 팔로워, 참여율 등)로 변환 후 한 디렉터리에 모아 학습 데이터로 사용.

---

## 6. 학습 데이터로 쓸 때 권장 필드

- **식별**: `platform`, `channel_id`, `username`, `profile_url`
- **규모**: `followers_count`, `subscribers_count`, `avg_views`
- **참여**: `engagement_rate`, `avg_likes`, `avg_comments`
- **분류/선별**: `content_categories` (분야, 해시태그 등)
- **메타**: `collected_at` (수집 시점)

이 필드들을 기준으로 “우리 사업에 맞는 인플루언서”를 필터링·랭킹하는 모델이나 룰을 설계할 수 있습니다.
